# 命运气象站 Fate Weather Station

> 《优秀数字艺术作品鉴赏》结课作业 - 非实体装置部分  
> 形式：移动端 H5 交互原型 + 多人实时后端 + 数据可视化 + 装置概念图 + 提交文档

## 1. 项目简介

「命运气象站」是一件围绕“虚拟体育彩票投注”和“概率可视化”展开的数字艺术作品。观众在 H5 页面中进入一个仿体彩计算器界面，选择比赛、玩法、比分、过关方式和倍数，完成一次不涉及真实金钱的虚拟投注。系统随后根据模拟开奖结果计算所有投注的累计投注额、累计返还额、净盈亏和返还率，并将这些数据映射到一个“气象球”的颜色、亮度和光晕强度上。

作品想表达的核心不是“预测比赛”，而是展示赔率结构如何把人的期待变成可计算的风险。赔率表看似中立，实则包含返奖率和抽水机制；当参与次数增加，个体的侥幸会逐渐被整体概率吞没。

## 2. 已完成内容

- 移动端 H5 页面：`index.html`
- 投注终端：胜平负、比分、总进球、半全场四种玩法
- 参考截图式比分网格：支持选择、取消、过关方式、倍数、投注额和理论最高奖计算
- 虚拟投注提示：明确标注不涉及真实购买、支付或兑奖
- 模拟开奖：主胜、平局、客胜、随机四种开奖方式
- 多人实时后端：所有用户投注、开奖和盈亏统一汇总
- 每日可投注赛程更新：服务器每天自动刷新一批可投注比赛
- The Odds API 接入：可用真实 bookmaker 胜平负赔率作为赔率来源
- football-data.org 接入：用于世界杯赛果、状态和真实比分结算
- 世界杯优先：默认配置为 `soccer_fifa_world_cup`，并按 `Asia/Shanghai` 显示开赛时间
- 调用额度保护：服务器集中请求、每日/月度预算、记录官方剩余额度
- 赔率实时更新：服务器每 45 秒对缓存赔率做小幅波动，前端自动同步，不额外消耗 API 次数
- 命运气象站：魔法球 SVG、全站实时数据面板、盈亏进度条、公式展示、历史轨迹
- 微信分享元信息：标题、描述和缩略图已写入页面
- 概念图：晴天、阴霾、风暴三种球体天气状态
- 提交文档：创作说明、提交清单、项目说明 PDF
- 页面截图：投注终端和气象站截图

## 3. 如何打开

### 本地离线演示

直接双击打开：

```text
fate-weather-station/index.html
```

如果浏览器限制本地文件读取图片，可以在项目文件夹中启动本地预览服务后访问：

```text
http://127.0.0.1:8765/
```

### 多人实时演示

进入项目目录运行：

```text
npm start
```

然后打开：

```text
http://127.0.0.1:8765/
```

顶部显示“多人实时版”时，说明页面已经连接服务器。此时气象球读取的是服务器中所有用户的总投注、总返还和总盈亏。

如果要在微信里给别人访问，需要把本项目部署到公网 HTTPS 地址。

## 4. 演示流程

1. 打开页面，默认进入「投注终端」。
2. 点击「一键选两场」可快速复现参考截图中的两场比分选择。
3. 点击「确认投注」完成一次虚拟投注。
4. 点击「模拟开奖」中的主胜、平局、客胜或随机。
5. 切换到「气象站」，观察魔法球颜色和数据面板变化。
6. 点击「自动模拟10轮」，快速展示返还率和净盈亏的长期趋势。

## 5. 数据映射公式

```text
净盈亏 = Σ(中奖金额) - Σ(投注金额)
返还率 = Σ(中奖金额) / Σ(投注金额) × 100%
色相 = 45° + (1 - 返还率) × 215°
亮度 = 25% + 返还率 × 45%
光晕强度 = |净盈亏| / 5000 × 100%
```

映射逻辑不人为调节结果，不预设输赢方向。页面上的颜色变化完全由虚拟投注和开奖后的盈亏数据计算得出。

## 7. 非装置部分说明

本次完成的是“非实体装置”部分：H5 交互、算法映射、数据可视化、概念图和课程提交材料。实体球体、灯带、控制器和底座暂未制作，但已在页面与文档中以概念图和参数说明的方式呈现。

## 8. 微信与多人实时说明

- 微信不能访问电脑本地 `file://` 页面，必须部署到公网 HTTPS。
- 页面已加入微信/社交平台可读取的标题、描述和缩略图信息。
- 服务器接口 `/api/state` 返回全站总盈亏，气象球使用该接口数据。
- 实时更新优先使用 SSE 推送；微信环境不支持时自动降级为 5 秒轮询。
- 当前可接入 The Odds API 获取真实 bookmaker 赔率。默认只抓 `soccer_fifa_world_cup`，避免其它联赛混入世界杯页面。作品仍为虚拟投注，不涉及真实交易。
- 如果要求世界杯赛程准确，请在 Render 环境变量中保持 `ODDS_API_SPORTS=soccer_fifa_world_cup` 和 `REQUIRE_REAL_MATCHES=true`；当接口没有返回世界杯比赛时，页面会显示“暂无真实世界杯赛程”，不会显示模拟赛程冒充世界杯。
- 如果要按真实比分开奖，请同时设置 `FOOTBALL_DATA_TOKEN` 和 `FOOTBALL_DATA_COMPETITIONS=WC`。
