## 2026-07-07 更新：微信公网与多人实时版

为确保作品可以在微信中打开、解析和分享，并且气象球能够反映**所有用户的总盈亏情况**，项目已从“纯本地 H5 原型”升级为“前端 H5 + 多人实时后端”的结构。

### 更新后的关键机制

1. **微信可访问/可解析**
   - 页面已加入标题、描述、关键词、`og:title`、`og:description`、`og:image` 和分享缩略图。
   - 作品需要部署到公网 HTTPS 地址后，微信才能稳定打开和解析；本地 `file://` 页面不能作为正式微信访问链接。

2. **每日可投注赛程与赔率更新**
   - 新增 `server.js` 后端服务。
   - 服务器统一维护比赛和赔率数据。
   - 当前版本支持 The Odds API 作为真实 bookmaker 赔率来源；如果未配置 key 或额度不足，则自动回退模拟数据。
   - The Odds API key 通过环境变量 `ODDS_API_KEY` 配置，不写入前端或 GitHub。
   - 赛果/比分通过 football-data.org 获取，key 通过环境变量 `FOOTBALL_DATA_TOKEN` 配置；世界杯比赛建议设置 `FOOTBALL_DATA_COMPETITIONS=WC`。
   - 服务器每天 08:00（服务器时间，可通过 `DAILY_REFRESH_HOUR` 调整）自动刷新一批当天/次日可投注比赛。
   - 为控制免费额度，默认每天最多使用 `ODDS_API_DAILY_LIMIT=8`，每月最多使用 `ODDS_API_MONTHLY_LIMIT=450`，并记录官方响应头 `x-requests-remaining / x-requests-used / x-requests-last`。
   - 服务器每 45 秒只对缓存赔率做本地小幅波动并推送给用户，不额外调用外部 API。
   - 用户下注时，服务器会以当前最新赔率重新校验并记录，避免不同用户页面赔率不同步。

3. **获奖情况实时更新**
   - 开奖不再只影响单个浏览器，而是提交到服务器统一结算。
   - 服务器结算全站待开奖投注后，会更新每个用户的中奖/未中状态、全站累计返还、净盈亏和历史轨迹。
   - 前端优先使用 SSE 实时推送；如果微信 WebView 不支持，则自动降级为 5 秒轮询。

4. **气象球读取全站总盈亏**
   - 气象球不再只读取当前浏览器本地数据，而是读取服务器接口 `/api/state`。
   - 核心字段包括：`totalBet`（全站累计投注）、`totalReturn`（全站累计返还）、`netProfit`（全站净盈亏）、`returnRate`（全站返还率）。
   - 因此，多个用户在微信中访问同一个链接时，气象球显示的是所有用户共同参与后的集体命运数据。

5. **离线兜底**
   - 如果没有运行服务器，页面仍可作为本地演示版打开。
   - 顶部状态会显示“本地演示版”；只有显示“多人实时版”时，才代表已接入全站数据。

## 一、项目概述

「命运气象站」是一个基于**虚拟体育彩票投注**与**物理气象装置**的跨媒介数字艺术作品。

用户通过 H5 页面进入一套高度仿真的体彩计算器，选择比赛、玩法、赔率、过关方式与投注金额，完成「虚拟投注」。比赛结果揭晓后，系统根据所有用户的**真实盈亏数据**（而非人为调参）驱动一个「魔法球」装置的颜色与天气状态，将抽象的数学概率转化为可感知的视觉气候。

作品的核心批判在于：**体育彩票的赔率结构本身就内含了庄家的抽水机制（返奖率通常 65%-73%），这意味着长期参与者的集体命运必然趋向亏损。装置的颜色变化不是创作者的主观操控，而是赔率数学的诚实呈现。**

## 二、核心概念与隐喻

### 2.1 三层隐喻结构

| 层级 | 现实对应 | 装置映射 | 隐喻 |
|------|----------|----------|------|
| **投注层** | 体彩计算器（胜平负、比分、过关、倍数） | H5 交互页面 | 参与者以为自己在「预测」 |
| **结果层** | 比赛胜负/比分结果 | 开奖逻辑 | 随机性制造的公平幻觉 |
| **命运层** | 盈亏金额（Σ中奖 - Σ投注） | 球体颜色/天气 | 数学宿命：久赌必亏 |

### 2.2 关键洞察

> **"赔率表不是预测工具，而是定价工具。"**

体彩的赔率由精算师设定，已经内含了庄家利润（返奖率 < 100%）。用户看到的「3.68」「1.78」等数字，不是对比赛结果的客观估计，而是对**用户行为分布**的定价。当足够多的用户参与后，无论单场结果如何随机，**集体的返还率必然收敛于返奖率上限**（约 73%），净盈亏必然为负。

装置用颜色诚实地展示了这个过程：
- 初期少量投注时，返还率可能因偶然波动而偏高 → **琥珀暖色**
- 随着参与人数增加，返还率向 70% 收敛 → **铅灰阴霾**
- 长期运行后，净盈亏持续为负 → **紫电风暴**

这不是悲观主义，是**数学必然**。

## 四、页面结构详解

### 4.1 Tab 1: 投注终端

#### 玩法导航
- **胜平负**：标准 3×1 赔率矩阵（胜/平/负）
- **比分**：1:0、2:1、3:2 等 25+ 种比分赔率网格
- **总进球**：0-7+ 球数赔率（预留）
- **半全场**：半场/全场结果组合（预留）

#### 比赛卡片
每张卡片包含：
- 联赛名称 + 比赛时间
- 主队 VS 客队
- 玩法赔率按钮（点击选中/取消）

#### 底部投注单
悬浮于页面底部，包含：
- **已选场数**：实时计数
- **过关方式**：动态生成（单关 / 2串1 / 3串1 / ... / n串1）
- **倍数输入**：1-99 倍
- **金额输入**：每注金额（默认 ¥10）
- **实时计算**：
  - 注数 = 过关组合数（单关时为场数，串关时为 1）
  - 总投注额 = 注数 × 倍数 × 金额
  - 理论最高奖 = 总投注额 × 最大赔率乘积
- **确认投注**：点击后记录投注数据，清空选择，显示成功提示

> ⚠️ **明确标注**：所有投注为虚拟交易，不涉及真实金钱。

#### 模拟开奖区
页面底部设有「模拟开奖」按钮组：
- 🎯 主胜 / ⚖️ 平局 / ❌ 客胜
- 点击后，系统对所有**未开奖**的投注进行结算
- 结算逻辑：根据用户选择的赔率与结果比对，计算中奖金额

### 4.2 Tab 2: 命运气象站

#### 魔法球（SVG 渲染）
- 使用 SVG `radialGradient` 实现 3 色渐变球体
- 顶部高光椭圆模拟玻璃反光
- `filter: drop-shadow()` 实现光晕效果
- 所有颜色参数通过 JS 实时计算更新

#### 数据面板（2×2 网格）
| 指标 | 说明 |
|------|------|
| 累计投注总额 | 所有用户虚拟投注金额之和 |
| 累计返还总额 | 所有开奖中奖金额之和 |
| 净盈亏 | 返还 - 投注（正为绿色，负为红色） |
| 返还率 | 返还 / 投注 × 100% |

#### 净盈亏可视化
- 以 0 为中心的双向进度条
- 右侧（正）：绿色渐变，表示净盈利
- 左侧（负）：红色渐变，表示净亏损
- 参考最大值：¥5000（用于归一化显示）

#### 映射公式展示
完全公开的计算逻辑，以代码风格呈现：

```
净盈亏 = Σ(中奖金额) - Σ(投注金额)
返还率 = Σ(中奖金额) / Σ(投注金额) × 100%
色相 = 45° + (1 - 返还率) × 215°
亮度 = 25% + 返还率 × 45%
光晕强度 = |净盈亏| / 参考最大值 × 100%
```

#### 历史轨迹
横向滚动的时间轴，显示最近 15 场开奖的：
- 单场盈亏金额（带 +/- 符号）
- 单场返还率百分比
- 颜色编码：盈利绿色 / 亏损红色 / 走水灰色

#### 装置设计图占位区
- 显示「实体装置概念设计」占位图
- 下方列出装置规格：球体、光源、控制器、底座、映射逻辑

## 五、映射算法（完全透明）

### 5.1 核心原则

**不人为调整胜负权重。不预设命运走向。让赔率结构自己说话。**

### 5.2 计算流程

```
Step 1: 用户完成虚拟投注
        → 记录：投注金额、选择的赔率、过关方式、倍数

Step 2: 比赛结果揭晓（模拟开奖）
        → 遍历所有未开奖投注
        → 判断：用户选择是否匹配结果？
           ├─ 匹配 → 中奖金额 = 投注额 × 赔率
           └─ 不匹配 → 中奖金额 = 0

Step 3: 更新全局统计
        → totalBet += 本轮投注额
        → totalReturn += 本轮中奖金额
        → netProfit = totalReturn - totalBet
        → returnRate = totalReturn / totalBet

Step 4: 映射到装置颜色
        → hue = 45 + (1 - returnRate) * 215
        → lightness = 25 + returnRate * 45
        → glowRadius = 20 + |netProfit| / MAX_REF * 40
        → 更新 SVG 渐变 stop-color
```

### 5.3 为什么长期趋向冷色？

**不是创作者调的，是数学事实：**

- 体彩返奖率上限约为 73%（即返还率 ≤ 0.73）
- 当 totalBet → ∞ 时，totalReturn / totalBet → 返奖率
- 因此 returnRate 长期收敛于 0.73 以下
- 代入色相公式：`hue = 45 + (1 - 0.73) × 215 = 45 + 58 = 103°`
- 103° 处于黄绿到灰蓝的过渡区，装置呈现冷色调

如果返奖率更低（如 65%），则 hue = 45 + 75 = 120°，偏向灰绿。

**装置的颜色变化是赔率结构的诚实镜像。**

## 六、数据模型

### 6.1 投注记录（Bet）

```typescript
interface Bet {
  id: number;              // 投注编号
  matchId: number;         // 比赛ID
  type: 'spf' | 'score' | 'goals' | 'half';  // 玩法类型
  value: string;           // 选择值（如 'win', '1:0'）
  odds: number;            // 赔率
  pass: number;            // 过关数（1=单关, 2=2串1...）
  times: number;           // 倍数
  amount: number;          // 每注金额
  totalMoney: number;      // 总投注额 = 注数 × times × amount
  settled: boolean;        // 是否已开奖
  winAmount?: number;       // 中奖金额（开奖后填充）
  profit?: number;          // 盈亏 = winAmount - totalMoney
}
```

### 6.2 开奖历史（HistoryEntry）

```typescript
interface HistoryEntry {
  result: 'win' | 'draw' | 'loss';  // 开奖结果
  bet: number;                      // 本轮投注额
  ret: number;                      // 本轮返还额
  profit: number;                   // 本轮盈亏
}
```

### 6.3 全局状态

```typescript
interface GlobalState {
  allBets: Bet[];           // 所有投注
  settledBets: Bet[];       // 已开奖
  totalBet: number;         // 累计投注
  totalReturn: number;      // 累计返还
  history: HistoryEntry[];   // 开奖历史
}
```

## 八、使用说明

### 8.1 演示流程

1. **打开页面** → 默认进入 Tab 1（投注终端）
2. **选择玩法** → 点击顶部「胜平负」「比分」等切换
3. **选择比赛** → 点击赔率按钮选中（再次点击取消）
4. **配置投注单** → 底部选择过关方式、倍数、金额
5. **确认投注** → 点击「确认投注（虚拟）」
6. **模拟开奖** → 点击「模拟开奖」按钮（主胜/平局/客胜）
7. **查看装置** → 切换到 Tab 2，观察球体颜色变化
8. **重复步骤 3-7** → 多次投注后观察返还率趋向

### 8.2 自动模拟

点击「自动模拟10场」按钮，系统将：
- 随机选择比赛和赔率
- 随机生成投注金额和倍数
- 随机开奖
- 连续运行 10 轮，快速展示长期趋向

### 8.3 重置

点击「重置所有数据」清空所有投注和开奖记录，装置恢复初始状态。

## 九、创作说明（课程提交用）

### 9.1 创作灵感来源

从体育彩票的「公正幻觉」出发。赔率表看似数学中立，实则精确计算了人性的贪婪与损失厌恶。课程中接触到的数字交互装置作品（如 teamLab 的集体交互）启发了我把「个体选择」汇聚为「集体视觉」。我注意到一个悖论：人们相信自己在「预测」比赛，实际上赔率早已预测了他们的行为分布。

### 9.2 使用的AI与数字工具

- **H5 前端**：Cursor / Claude 辅助生成 Vue 风格的组件代码，模拟体彩官方 UI
- **视觉设计**：Midjourney 生成「魔法球装置」概念图、天气氛围参考
- **算法逻辑**：ChatGPT 辅助验证返还率收敛的数学模型
- **数据可视化**：手动实现 SVG 渐变映射，验证 HSL 色相公式的连续性

### 9.3 作品解释与说明

「命运气象站」由虚拟投注终端和物理气象装置组成。用户在前端「假装买球」，他们的选择被抽象为数据流。比赛结果揭晓后，系统根据**真实的盈亏金额**驱动装置颜色——不是创作者预设的情绪，而是赔率结构的数学必然。

核心设计在于**完全透明的映射公式**：
- 净盈亏 = Σ中奖 - Σ投注
- 色相 = 45° + (1 - 返还率) × 215°

返还率长期收敛于体彩返奖率上限（约 73%），因此装置色相长期收敛于 103°（灰蓝区）。这不是悲观主义，是数学诚实。作品试图让观众在「玩」的过程中，无意识体验到「久赌必亏」的宿命感——但这不是说教，是装置颜色的自然演化。

### 9.4 对数字共创过程的评价

AI 工具极大地加速了从概念到原型的过程，尤其是前端 UI 和数学验证。但核心的批判性思维——「为什么映射算法要这样设计」「为什么必须透明」——必须由创作者自己完成。装置艺术的价值不在技术复杂度，而在概念是否锋利。这个作品试图用**数学的不可辩驳性**替代**道德的说教性**，让赔率结构自己成为批判者。
