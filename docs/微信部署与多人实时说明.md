# 微信部署与多人实时说明

## 1. 关键结论

如果希望微信可以正常打开、解析和分享网页，并且让气象球读取“所有用户”的总盈亏，不能再使用 `file://` 本地文件方式打开页面。必须把项目部署到一个公网 HTTPS 地址，例如：

```text
https://你的域名/
```

本项目已升级为“前端 + 多人实时后端”结构：

- 微信用户访问同一个网页地址。
- 每个用户的虚拟投注会提交到同一个服务器。
- 服务器统一保存所有投注、开奖、返还和盈亏。
- 气象球读取服务器返回的全站累计投注、累计返还、净盈亏和返还率。
- 赔率由服务器统一更新，所有用户页面会同步刷新。
- 开奖后，获奖情况和全站盈亏会实时推送；不支持推送的微信环境会自动每 5 秒轮询一次。

## 2. 本地测试方式

进入项目目录：

```text
cd fate-weather-station
node server.js
```

然后打开：

```text
http://127.0.0.1:8765/
```

注意：本地地址只能自己电脑测试，微信好友无法访问。要给微信访问，必须部署到公网。

## 3. 部署方式建议

### 方案 A：Render / Railway / Fly.io

这些平台可以直接运行 `node server.js`。

部署时设置：

```text
Start Command: npm start
Port: 使用平台自动分配的 PORT
```

项目已包含 `package.json`，平台会自动识别 Node 项目。

### 方案 B：自己的云服务器

服务器需要安装 Node.js 18 或更高版本。

运行：

```text
npm start
```

如果要长期运行，可以用 PM2：

```text
pm2 start server.js --name fate-weather-station
```

再用 Nginx 配置 HTTPS 反向代理到 `127.0.0.1:8765`。

## 4. 微信可解析网页需要什么

本项目已在 `index.html` 中加入：

- 页面标题
- 页面描述
- 关键词
- `og:title`
- `og:description`
- `og:image`
- 微信/社交分享缩略图 `assets/wx-share-thumb.jpg`

但微信里更稳定的分享卡片通常还需要：

1. 网页必须是公网 HTTPS。
2. 图片必须可以公网访问。
3. 如果要自定义微信分享标题、缩略图和描述，通常需要微信公众号 JS-SDK：
   - 已认证公众号或开放平台能力；
   - 配置 JS 接口安全域名；
   - 后端生成 `wx.config` 签名。

本项目目前已满足普通网页解析和分享元信息准备；如果课程提交只是发链接让老师在微信中打开，部署到 HTTPS 公网即可。

## 5. 赔率实时更新说明

当前版本已支持接入 **The Odds API** 获取真实 bookmaker 胜平负赔率。为了保护免费额度，前端用户不会直接请求 The Odds API，只有服务器会集中请求并缓存。

如果没有配置 `ODDS_API_KEY`，系统会自动回退为模拟赔率。

服务器每 45 秒的赔率变化不是请求外部 API，而是对已缓存赔率做本地小幅波动；这样不会消耗 The Odds API 额度。真正消耗额度的操作只有：

- 每天刷新可投注赛程/赔率；
- 手动请求 `POST /api/admin/odds-api`；
- 部署启动后需要刷新当天赛程时。

## 5.1 每日可投注赛程更新

当前版本已加入“每日自动刷新可投注比赛”的机制：

- 服务器启动后会检查当天日期。
- 每天到达设定时间后，优先从 The Odds API 获取真实赔率赛程。
- 如果 The Odds API 未配置、额度不足或请求失败，则回退为 football-data.org 赛程或模拟赛程。
- 默认刷新时间为服务器时间每天 08:00。
- 可通过环境变量调整：

```text
DAILY_REFRESH_HOUR=8
```

注意：The Odds API 提供的是 bookmaker 赔率，不是中国体彩官方赔率；作品仍然是虚拟投注和艺术演示，不涉及真实交易。

## 5.2 The Odds API 调用额度控制

本项目有三层保护，避免超过免费额度：

1. **服务器集中请求**
   - 所有微信用户都只访问你的服务器；
   - 不会出现“每个用户打开页面就请求一次 The Odds API”的情况。

2. **本地每日/月度预算**
   - 默认每日最多使用：

```text
ODDS_API_DAILY_LIMIT=8
```

   - 默认每月最多使用：

```text
ODDS_API_MONTHLY_LIMIT=450
```

   - 这样即使免费额度是 500 credits/month，也会留下安全余量。

3. **读取官方响应头**
   - The Odds API 返回的响应头会被记录：

```text
x-requests-remaining
x-requests-used
x-requests-last
```

   - 如果官方剩余额度不足，服务器会停止继续调用并回退缓存/模拟数据。

推荐部署环境变量：

```text
ODDS_API_KEY=你的 The Odds API key
ODDS_API_DAILY_LIMIT=8
ODDS_API_MONTHLY_LIMIT=450
ODDS_API_REGIONS=eu
ODDS_API_MARKETS=h2h
ODDS_API_MAX_SPORTS_PER_REFRESH=3
```

其中 `h2h` 对应胜平负/独赢市场；`eu` 对应欧洲地区 bookmaker。

如果演示时想手动刷新今天的可投注比赛，可以请求：

```text
POST /api/admin/matches
```

如果以后接入更多真实数据，可以继续扩展 `server.js` 中的：

```text
refreshDailyMatches()
updateOddsTick()
```

把它改成请求真实赛程/赔率接口，然后更新 `db.matches` 和 `db.options`。

## 6. 获奖情况实时更新说明

点击页面中的“主胜 / 平局 / 客胜 / 随机”开奖按钮后，服务器会结算全站所有待开奖投注。结算完成后：

- 每个用户的“我的投注记录”会更新中奖/未中状态。
- 全站累计返还额更新。
- 全站净盈亏更新。
- 气象球颜色、亮度和光晕同步更新。
- 历史轨迹追加一条开奖记录。

## 7. 气象球数据来源

气象球不再读取单个浏览器的本地数据，而是读取服务器接口：

```text
GET /api/state
```

其中关键字段为：

```text
totalBet       全站累计投注
totalReturn    全站累计返还
netProfit      全站净盈亏
returnRate     全站返还率
history        全站最近开奖轨迹
```

因此，只要所有观众访问同一个服务器地址，气象球展示的就是所有用户共同参与后的总盈亏情况。

## 8. 离线兜底

如果直接双击 `index.html`，或部署环境没有运行 `server.js`，页面会自动退回“本地演示版”。本地演示版可以展示交互，但数据只保存在当前浏览器中，不代表所有用户。

顶部状态会显示：

```text
多人实时版 / 本地演示版
```

用它可以判断当前是否真的接入了全站数据。
